## ⏰ 第五步：修改运行频率

默认每5分钟运行一次，如需修改：

打开 `.github/workflows/stock-monitor.yml`，找到：

```yaml
schedule:
  - cron: '*/5 * * * *'  # 每5分钟
```

修改为其他频率：

| 频率 | Cron表达式 | 说明 |
|------|-----------|------|
| 每5分钟 | `*/5 * * * *` | 默认 |
| 每10分钟 | `*/10 * * * *` | 适中 |
| 每30分钟 | `*/30 * * * *` | 节省资源 |
| 每小时 | `0 * * * *` | 低频 |
| 交易时间段 | 自定义 | 见下方 |

### 仅在交易时间运行

```yaml
schedule:
  # 工作日上午9:30-11:30，下午13:00-15:00
  - cron: '30-59 9-11 * * 1-5'   # 上午
  - cron: '0-59 13-15 * * 1-5'    # 下午
```

## ❓ 常见问题

### Q1: Actions没有自动运行？

**检查**：
1. 确认workflow文件在 `.github/workflows/` 目录下
2. 确认文件名以 `.yml` 或 `.yaml` 结尾
3. 确认cron表达式格式正确
4. 查看Actions页面是否有错误信息

### Q2: 获取股价失败？

**可能原因**：
- 网络问题：GitHub服务器可能无法访问某些API
- API限制：频繁访问可能被限制
- 时区问题：检查cron表达式是否正确（使用UTC时间）

**解决**：
- 查看详细日志
- 尝试更换数据源
- 增加运行间隔

### Q3: 如何查看历史数据？

**方法**：
1. 查看Actions运行历史
2. 下载artifact（日志文件）
3. 配置日志上传到其他存储

## 📊 数据存储说明

### GitHub Actions中的数据

**临时数据**：
- 每次运行时数据存在内存中
- 运行结束后自动清除

**持久化数据**：
- 日志文件：通过artifact保存30天
- 提醒记录：同样通过artifact保存

### 长期存储方案

如需长期保存数据，可以考虑：

1. **配置GitHub Pages** - 展示数据可视化
2. **上传到外部存储** - 如云数据库
3. **使用GitHub仓库** - 每次运行提交数据
